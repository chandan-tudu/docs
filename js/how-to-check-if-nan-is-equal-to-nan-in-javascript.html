How To Check If NaN Is Equal To NaN In JavaScript

JavaScript's handling of NaN (Not a Number) can be a source of confusion, especially when it comes to equality comparisons.

In this article, we'll delve into the peculiarities of NaN and explore different ways to check if NaN is equal to NaN in JavaScript.

The NaN Conundrum:

NaN represents the result of an operation that cannot produce a meaningful result, such as dividing zero by zero.

However, NaN has some unique characteristics that make it tricky to work with, particularly in equality comparisons.

const result = 0 / 0;
console.log(result); // Outputs: NaN

The surprising aspect is that NaN is not equal to itself, unlike other values.

console.log(NaN === NaN); // Outputs: false

1. Using isNaN() Function:

JavaScript provides the isNaN() function to specifically check for NaN values.

const result = 0 / 0;
console.log(isNaN(result)); // Outputs: true

However, isNaN() has a quirkâ€”it also returns true for values that are not actually NaN but are not coercible to a numeric value.

console.log(isNaN('hello')); // Outputs: true

To address this, you can use Number.isNaN(), introduced in ECMAScript 2015 (ES6), which only returns true for actual NaN values.

console.log(Number.isNaN(result)); // Outputs: true
console.log(Number.isNaN('hello')); // Outputs: false

2. Using a Custom Equality Function:

To check if NaN is equal to NaN without relying on isNaN() or Number.isNaN(), you can create a custom equality function.

function isNaNCustom(value) {
    return value !== value;
}

const result = 0 / 0;
console.log(isNaNCustom(result)); // Outputs: true

This method exploits the fact that NaN is the only JavaScript value not equal to itself, making it suitable for a custom isNaN check.

3. Using Object.is():

The Object.is() method was introduced in ECMAScript 2015 (ES6) to provide a more robust equality comparison, including handling NaN.

const result = 0 / 0;
console.log(Object.is(result, NaN)); // Outputs: true

Object.is() behaves similarly to strict equality (===) but with a few key differences, such as treating -0 and +0 as unequal and correctly handling NaN comparisons.

Considerations:

  1. Use of isNaN(): Be cautious when using isNaN() for NaN checks, as it may return unexpected results for non-numeric values.

  2. Compatibility: If you need to support older JavaScript environments, be aware that some methods, such as Number.isNaN() and Object.is(), might not be available.

  3. Semantic Differences: Understand the semantic differences between isNaN() and Object.is() when choosing an approach. Object.is() is more predictable in certain scenarios.

  4. Readability: Choose the method that aligns with the readability goals of your codebase and your specific NaN-checking requirements.

Conclusion:

NaN in JavaScript introduces some quirks when it comes to equality comparisons. While isNaN() and Number.isNaN() provide dedicated methods for NaN checks, developers should be aware of their limitations.

Creating a custom equality function or using Object.is() offers alternative approaches, each with its own trade-offs.

By understanding the nuances of NaN and choosing an appropriate method, developers can navigate the complexities of NaN equality comparisons in JavaScript effectively.
