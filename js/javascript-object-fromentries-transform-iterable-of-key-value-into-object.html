JavaScript Object.fromentries(): Transform Iterable Of Key-value Into Object

In the realm of JavaScript, the language constantly evolves to provide developers with more expressive and efficient tools. One such addition in ECMAScript 2019 (ES10) is the Object.fromEntries() method.

This method serves as a counterpart to Object.entries() and offers a convenient way to create an object from an iterable of key-value pairs.

In this article, we will explore the intricacies of Object.fromEntries() and understand how it enhances object manipulation in JavaScript.

The Basics of Object.fromEntries()

The primary purpose of Object.fromEntries() is to transform an iterable of key-value pairs into an object. The iterable can be an array, a Map, or any other data structure that supports the key-value pair structure.

Here is a simple example using an array of key-value pairs:

const keyValuePairs = [
    ['name', 'John'],
    ['age', 30],
    ['city', 'New York'],
];

const myObject = Object.fromEntries(keyValuePairs);
console.log(myObject);
// Output: { name: 'John', age: 30, city: 'New York' }

In this example, Object.fromEntries() takes an array of key-value pairs and constructs an object with properties corresponding to each pair.

Use Cases and Practical Applications

1. Converting Map to Object:

Object.fromEntries() is particularly useful when working with Map objects. While Maps and objects share similarities, converting between them was not as straightforward before the introduction of this method.

const myMap = new Map([
    ['fruit', 'apple'],
    ['color', 'red'],
    ['quantity', 3],
]);

const myObject = Object.fromEntries(myMap);
console.log(myObject);
// Output: { fruit: 'apple', color: 'red', quantity: 3 }

Now, with a single line of code, you can effortlessly convert a Map into an object.

2. Filtering and Transforming Object Properties:

Suppose you have an object, and you want to create a new object by filtering or transforming its properties.

Object.entries() combined with Object.fromEntries() provides an elegant solution.

const originalObject = {
    name: 'Alice',
    age: 25,
    country: 'Wonderland',
    isAdmin: true,
};

const filteredObject = Object.fromEntries(
    Object.entries(originalObject).filter(([key, value]) => key !== 'isAdmin')
);

console.log(filteredObject);
// Output: { name: 'Alice', age: 25, country: 'Wonderland' }

In this example, properties with the key 'isAdmin' are filtered out, creating a new object without that property.

3. Parsing Query Parameters:

When dealing with URL query parameters, Object.fromEntries() can be used to convert an array of key-value pairs into an object.

const queryString = 'name=John&age=30&city=New York';

const queryParams = Object.fromEntries(
    new URLSearchParams(queryString).entries()
);

console.log(queryParams);
// Output: { name: 'John', age: '30', city: 'New York' }

Here, URLSearchParams is used to parse the query string, and Object.fromEntries() converts it into an object.

Conclusion

Object.fromEntries() is a versatile addition to JavaScript, offering a concise and expressive way to create objects from iterables of key-value pairs.

Whether you're transforming Maps, filtering object properties, or parsing query parameters, this method streamlines the code and enhances readability.

As the JavaScript language continues to evolve, understanding and leveraging new features like Object.fromEntries() contributes to writing more efficient and maintainable code.
