How To Check For NaN In JavaScript

Dealing with NaN (Not a Number) is a common task in JavaScript, especially when working with numeric operations. JavaScript provides specific methods and practices to check whether a value is NaN.

In this article, we'll explore these methods and discuss best practices for handling NaN checks.

1. Using isNaN() Function

The most straightforward way to check for NaN in JavaScript is by using the built-in isNaN() function.

const myValue = 42;

if (isNaN(myValue)) {
    console.log('It is NaN!');
} else {
    console.log('It is not NaN.');
}

The isNaN() function returns true if the provided value is NaN; otherwise, it returns false.

It's important to note that isNaN() is not exclusive to checking for NaN; it also returns true for other non-numeric values.

console.log(isNaN('Hello')); // Output: true

To overcome this limitation, you can use the Number.isNaN() method introduced in ECMAScript 6 (ES6).

2. Using Number.isNaN()

The Number.isNaN() method is a more robust alternative to isNaN() for specifically checking NaN values.

const myValue = 42;

if (Number.isNaN(myValue)) {
    console.log('It is NaN!');
} else {
    console.log('It is not NaN.');
}

Unlike isNaN(), Number.isNaN() does not perform any type coercion and only returns true if the provided value is exactly NaN.

console.log(Number.isNaN('Hello')); // Output: false

3. Using Strict Equality (===)

Another way to check for NaN is by using strict equality (===) with itself. Since NaN is the only value not equal to itself, this method works.

const myValue = 42;

if (myValue !== myValue) {
    console.log('It is NaN!');
} else {
    console.log('It is not NaN.');
}

This method, while effective, is less commonly used compared to isNaN() or Number.isNaN().

Best Practices

  • Prefer Number.isNaN(): When specifically checking for NaN, Number.isNaN() is the recommended method as it avoids type coercion.

  • Be Aware of isNaN() Caveats: If using isNaN(), be cautious of its behavior with non-numeric values and consider whether it fits your use case.

  • Use Strict Equality for Simplicity: While less common, using strict equality (===) with itself is a simple way to check for NaN.

Conclusion

Checking for NaN in JavaScript is a crucial aspect of handling numeric operations. The choice between isNaN(), Number.isNaN(), or strict equality (===) depends on the specific use case and desired behavior.

By understanding the differences between these methods and following best practices, you can confidently handle NaN checks in your JavaScript code.
